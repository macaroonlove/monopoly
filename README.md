## 🎯 Project Goals
* **역사 교육의 대중화**: 딱딱한 한국사를 보드게임 형식으로 풀어내어 재미있게 학습할 수 있는 환경 제공.
* **로직의 완전 구현**: 모노폴리의 복잡한 구매, 건설, 통행료 계산 시스템을 100% 코드로 구현.
* **오프라인 멀티플레이**: PC 한 대를 활용해 최대 4인이 번갈아 가며 즐길 수 있는 로컬 멀티플레이 환경 구축.

---

### 🔹 Core Game Logic
* **[부동산 시스템]**: 각 지역의 대지 구매, 건물 건설 단계별 로직 구현.
* **[통행료 자동 계산]**: 소유주와 건설 단계에 따른 통행료 차감 및 자산 이동 시스템 구축.
* **[주사위 및 턴 제어]**: 유저별 턴 순서 제어 및 이동 로직.

### 🔹 UI & Interaction
* **[동적 오브젝트 제어]**: 건설 단계에 따라 깃발, 땅, 랜드마크 오브젝트를 실시간으로 `SetActive` 제어.
* **[로컬 멀티플레이 UI]**: 현재 턴인 플레이어의 상태를 직관적으로 보여주는 UI 시스템 제작.

## ※ Retrospective
> 💡 **Why Unity?**
* 당시 경진대회의 많은 팀이 기존에 오픈소스로 공개된 Python 기반의 모노폴리 프로젝트를 가져와 기능을 추가하는 안정적인 방식을 택했습니다.<br>하지만 저희 팀은 사용자에게 시각적으로 더 입체적이고 생동감 있는 경험을 제공하고 싶었습니다.<br>학우들과 교수님들께 단순한 '숫자 노름'이 아닌, 진짜 '게임'을 선사하며 분위기를 환기시키고자 하는 목표가 있었습니다.

> 🚀 **무엇을 배웠나? (유저와의 상호작용)**
* 익숙하지 않은 유니티를 선택해 맨땅에 헤딩하듯 개발을 시작했습니다.<br>**직관적인 피드백**: 코드가 화면 속 오브젝트의 움직임(주사위 굴리기, 건물 세우기 등)으로 즉각 나타나는 것을 보며, 유저와 실시간으로 상호작용하는 게임 개발의 매력에 깊이 빠지게 되었습니다.<br>**로직 구현의 집요함**: 비록 '한글 변수명'과 '수많은 if문'이 난무하는 미숙한 코드였지만, 복잡한 보드게임의 규칙을 하나하나 내 손으로 직접 설계하고 구현해내는 과정에서 개발자로서의 근성을 길렀습니다.

> ✨ **결과와 성장**
* 이 도전적인 선택은 결국 교내 경진대회 우수상이라는 성과로 이어졌습니다. 이 프로젝트는 저에게 단순히 '게임을 만드는 법'을 알려준 것이 아니라,<br>**사용자에게 즐거움을 주기 위해 더 나은 기술적 선택을 고민하는 태도**를 심어준 소중한 첫 시작이었습니다.
